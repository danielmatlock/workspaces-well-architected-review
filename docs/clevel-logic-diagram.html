C-Level Executive Briefing — Generation Logic

Gold-standard curated deck | 13 fixed slides | AI-powered curation via Bedrock Claude Haiku 4.5

Generation Pipeline

1
User Initiates
User clicks "C-Level Deck" → modal with optional grounding reports (So What + WAFR) and notification email
Frontend: exportCLevelDeck() → modal with dropdowns
▼
2
Fetch Grounding Context
Download selected S3 reports, strip HTML tags, truncate to 15,000 chars per report. Provides factual basis to reduce AI hallucinations.
Lambda: getReport → base64 decode → HTML strip
▼
3
Generate Tailored Recommendations
Batch all questions with notes into groups of 5. Send each batch to Bedrock for observation rewrite + recommendation generation. Sequential batching to stay within API Gateway 29s timeout.
Lambda: generate (×N batches) → Bedrock (4096 tokens each) → merge results
▼
4
AI Curation (curateDeck)
Send ALL findings (IDs, pillars, scores, notes, observations, recommendations) + pillar scores + grounding context to Bedrock in a single call. AI curates into business-framed Must/Should/Could tiers — NOT a mechanical RAG mapping.
Lambda: curateDeck → Bedrock (8192 tokens) → structured JSON response
▼
5
Build PowerPoint
buildCuratedCLevelPptx consumes the curated JSON and builds 13 fixed slides with visual design system (coloured spines, RAG bars, KPI cards, effort badges, callout boxes).
Frontend: pptxgenjs → base64 → S3 save via Lambda saveReport
▼
6
Notify & Complete
Send email notification (if email provided). Show success alert. Deck accessible via Saved Reports.
Lambda: notify → SES | Fallback: buildCLevelPptx (mechanical) if curation fails

AI Curation Rules

The AI is instructed to:

  • NOT map mechanically — RAG is an input to tiering, not the tiering itself
  • Select exactly 4 Critical Findings — board-level risks with business headlines + consequence text
  • Curate MUST (3-5 items) — genuine board-level risks + true quick wins
  • Curate SHOULD (4-6 items) — highest-value items to resource this quarter
  • Curate COULD (4-6 items) — strategic, longer-term items
  • NEVER use "customer" / "the customer" / "Customer-1" in any text — use org name or "the estate / the fleet"
  • NEVER present fully-implemented items as gaps or quick wins
  • Only quantify cost savings where source data provides figures — otherwise flag "needs more research"
  • Executive summary: exactly 3-4 short sentences (max 30 words each)
  • All headlines: 5-8 words, business-framed language

Fallback Behaviour

If curateDeck Lambda call fails:

  • Frontend logs error to console
  • Falls back to buildCLevelPptx (mechanical layout: RED→MUST, AMBER→SHOULD, GREEN→COULD)
  • Shows alert: "AI curation was unavailable — deck generated using standard layout"
  • Mechanical deck still saves to S3 and sends notification if email provided
